### Fixed
- #206: reconciled live-source contract, demo runbook, and implementation-status wording after final cache-wiring closure — the docs now describe the completed no-live-network source-client boundary accurately instead of saying #206 is still in progress.
- #81: `route_verified()` now binds **every** descriptor-derived authorization field, not just operation/handler id. `OperationDescriptor::authorization_fingerprint()` (`server/src/manifest.rs`) is a deterministic SHA-256 over canonical length-prefixed bytes of version/opcode/operation/payload_schema/target_kind/required_credentials/required_capabilities/accepted_evidence/replay_scope/max_ttl_seconds/handler_id/dev_binding/range (declared Vec order; no map iteration, no Debug formatting; no payload or evidence material). The verifier stamps it onto `VerifiedCallContext` at verification time (`descriptor_fingerprint`, schema bumped to `VERIFIED_CALL_CONTEXT_SCHEMA_VERSION = 2`), and `signed_context_matches_active_manifest()` re-checks it against the active manifest — plus rejects an actual TTL span (`expires_at - issued_at`) over the active `max_ttl_seconds` — before any route side effect. A trusted verifier can no longer sign a context whose operation/handler id match the active descriptor while its credentials/capabilities/evidence/schema/target kind/replay scope/TTL/range are stale: the RED matrix in `server/tests/gateway_layout.rs` proves a mutated-variant manifest yields exactly one inspectable `descriptor_context_mismatch` reject with zero handler calls, telemetry, replay reservations, or receipts. Empty fingerprints never route. Boundary: this hardens descriptor-derived authorization binding only — it is not evidence-aware live ingress (#78/#79) and not Dregg authority (#73).
